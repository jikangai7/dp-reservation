# セットアップ手順書（別端末への導入）

対象読者：**このツールをまだ使ったことがない端末に、新しく導入する方**。
導入が終わったら、日常的な使い方は [操作手順書.md](操作手順書.md) を参照してください。

---

## 前提条件

- Windows PC
- インターネット接続（初回のみ。Node.js・依存パッケージ・Chromiumのダウンロードに必要）
- Node.js（LTS版を推奨。動作確認は v26）

---

## 手順1：フォルダを転送する

`dp-reservation` フォルダを新しい端末にコピーしてください。

- **`node_modules` フォルダはコピー不要です**（サイズが大きく、OS依存のため）。
  コピー元に `node_modules` が含まれていても無視して構いません。
- 置き場所はどこでも構いません（例：`C:\work\dp-reservation`）。
  スクリプトはフォルダ内の相対位置を基準に動くため、置き場所を変えても問題ありません。
- `transfer.zip` のような圧縮ファイルで受け取った場合は、展開してから使用してください。

---

## 手順2：Node.jsを用意する

コマンドプロンプト（またはPowerShell）で以下を実行し、バージョンが表示されるか確認します。

```powershell
node -v
```

- **表示された場合**：v18以上であればそのまま次へ進んでください。
- **「認識されません」等のエラーが出た場合**：
  1. [https://nodejs.org](https://nodejs.org) から **LTS版**をダウンロードしてインストール
  2. インストール後、新しいコマンドプロンプト／PowerShellを開き直して再度 `node -v` を確認
  3. インストール済みのはずなのに認識されない場合は、以下でその場しのぎが可能です
     （インストール先が異なる場合はパスを読み替えてください）：
     ```powershell
     $env:Path = "C:\Program Files\nodejs;" + $env:Path
     ```

---

## 手順3：依存関係をインストールする

### 自動で行う場合（推奨）

`dp-reservation` フォルダ内の **`setup.bat`** をダブルクリックしてください。
Node.jsの確認・依存パッケージのインストール・ブラウザ（Chromium）のインストールを自動で行います。
完了まで数分かかります（回線速度による）。

### 手動で行う場合

```powershell
cd <dp-reservationフォルダのパス>
npm install
npx playwright install chromium
```

---

## 手順4：動作確認する

以下のコマンドを実行してください。

```powershell
node reserve.js OSA MNL 2026/10/01 2026/10/10
```

以下のように `REACHED: .../dp/reservation/input?...` が出力され、
コマンドが正常終了（追加のエラー表示なし）すればセットアップ完了です。

```
REACHED: https://www.skygate.co.jp/dp/reservation/input?...
  title    : 旅行者情報入力｜エアトリ
  packageId: packageId_...
  __tx__   : ...
```

より詳しい合否判定・トラブルシューティングは [検証手順.md](検証手順.md) を参照してください。

---

## 手順5：Web UI（ブラウザ画面）の起動確認

`dp-reservation` フォルダ内の **`start-server.bat`** をダブルクリックし、
ブラウザで `http://localhost:5178/` を開いて画面が表示されることを確認してください。

日常的な使い方は [操作手順書.md](操作手順書.md) を参照してください。

## よくあるトラブル

| 症状 | 原因の見当 | 対処 |
|---|---|---|
| `node : 用語 ... 認識されません` | Node.jsがPATHに無い | 手順2を再確認。`$env:Path` の一時追加、またはNode.js再インストール |
| `npm install` でエラーが出る | インターネット接続不良／社内プロキシ制限 | 接続環境を確認。プロキシ環境下では別途プロキシ設定が必要な場合あり |
| `npx playwright install chromium` が失敗する | 同上、またはディスク容量不足 | 接続・空き容量を確認して再実行 |
| `node reserve.js ...` がエラーで終了する | 依存インストールが未完了、またはサイト側の一時的な問題 | `setup.bat` を再実行。時間をおいて再試行 |
| ポート5178が使用中と出る | 別プロセスがすでにポートを使っている | `stop-server.bat` を実行してから `start-server.bat` を再実行 |

さらに詳しい技術情報は [README.md](README.md)・[仕様書.md](仕様書.md)・[引き継ぎ.md](引き継ぎ.md) を参照してください。
