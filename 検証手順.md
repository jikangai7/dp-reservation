# 検証手順（reserve.js が /dp/reservation/input へ到達することの確認）

4項目から予約者情報入力画面まで自動到達できることを、誰でも再現できる形で確認するための手順です。
所要時間：初回（依存導入込み）約5分、2回目以降 約1分。

> ゴール：`REACHED: .../dp/reservation/input?...` が出力され、終了コード0になること。
> 予約確定・決済は行いません（情報入力画面の表示到達までが確認対象）。

## STEP 0. 前提確認

PowerShell を開き、Node.js が使えることを確認します。

```powershell
# Node が PATH にない環境向け（既に通っていれば不要）
$env:Path = "C:\Program Files\nodejs;" + $env:Path
```

- [ ] `node -v` がバージョンを表示する
  - 表示されない場合：Node.js をインストールするか、`node.exe` のあるフォルダを上記 `$env:Path` に指定する。

## STEP 6.（任意）キー抽出と /dp/list 中間作業モード

`keys` / `open-list` / `proceed` を検証します。airCacheKey は失効しうるため、
**抽出したら間を置かず**次の手順で使ってください（下記は1つのPowerShellで連続実行）。

```powershell
# 6-1) 本物のキーを取得
$out = node reserve.js keys OSA MNL 2026/10/01 2026/10/10 2>&1 | Out-String
$k = ([regex]::Match($out, '(?s)\{.*\}').Value) | ConvertFrom-Json
"air=$($k.airCacheKey)"; "hotel=$($k.hotelCacheKey)"; "item=$($k.selectedItemKey)"

# 6-2) selectedItemKey 無しで一覧を開き、自動生成を確認
node reserve.js open-list --air $k.airCacheKey --hotel $k.hotelCacheKey

# 6-3) list-url 指定で予約入力まで進む
node reserve.js proceed --list-url $k.listUrl
"EXITCODE: $LASTEXITCODE"
```

- [ ] 6-1: JSONに `airCacheKey`/`hotelCacheKey`/`selectedItemKey`/`listUrl` が出る
- [ ] 6-1: `hotelCacheKey MATCH : YES`（生成値＝サーバー値）
- [ ] 6-2: `AUTO-GENERATED : YES → ..._dpci`（selectedItemKey無しでもサーバーが自動選択）
- [ ] 6-3: `REACHED: .../dp/reservation/input` が出て終了コード0
- [ ] （任意）`proceed --air $k.airCacheKey --hotel $k.hotelCacheKey`（itemなし）でも到達する

> PowerShell注意：`airCacheKey` はカンマを含むため、変数（`$k.airCacheKey`）経由か
> ダブルクオートで渡すこと。裸で渡すと配列に分割されて壊れる。

## トラブルシューティング

| 症状 | 原因の見当 | 対処 |
|---|---|---|
| `node : 用語 ... 認識されません` | Node が PATH に無い | STEP 0 の `$env:Path = "C:\Program Files\nodejs;" + $env:Path` を実行 |
| `Executable doesn't exist ... chromium` | ブラウザ未導入 | `npx playwright install chromium` を実行 |
| `FAILED at URL: .../dp/list...`（クリック後に進まない） | 「予約確認へ進む」セレクタ不一致 or avail遅延 | `--headed` で目視。ボタン文言・レイアウト変更ならセレクタ調整 |
| `FAILED at URL: .../account.airtrip.jp/auth...` | 匿名SSOがログイン要求に化けた | 時間をおいて再試行。継続する場合はトップ `/dp/` のフォーム入力方式に切替 |
| 待機タイムアウト | ネットワーク遅延／availが3秒以上 | 再実行。恒常的なら reserve.js の `timeout: 60000` を延長 |

**このステップで発見・修正した不具合**：`app.js` の `runAvailCheck`/`renderCards` が
`cardsEl.children[index]` でカードのDOM要素を取得していたため、グループ見出し（`<h2>`）を
`#cards` に挿入したことでインデックスがずれ、2グループ目以降のカードで「avail実行」が
誤ったDOM要素（見出し要素など）を操作しようとして失敗する不具合があった。各エントリに
実際のカード要素への参照（`entries[i].card`）を保持する方式に修正し解消。
