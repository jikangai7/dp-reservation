# 海外DP自動検証ツール 説明資料

*対象読者：上司（管理者）・エンジニア　/　作成日：2026年7月5日*

## 1. 概要（サマリー）

本ツールは、エアトリの海外DP（Dynamic Package＝航空券＋ホテルのセット商品）について、
指定した条件（出発地・目的地・日程）で実際に販売されているフライト・ホテルの組み合わせを自動で収集し、
**本当に予約可能か（在庫があり、予約者情報入力画面まで到達できるか）を自動検証する社内ツール**です。

手作業で1件ずつサイトを操作して確認していた作業を自動化し、複数の航空会社・複数のホテルについて
まとめて検証できるようにしています。予約の確定・決済までは行わず、**「予約直前まで到達できるか」の検証に限定**しています。

**現状のステータス：**基本機能・複数キャリア対応・ローカルWeb画面まで実装・動作確認済み。
現在も検証観点の拡充を継続中です。

## 2. 背景・目的

- DP商品は検索結果に表示されていても、実際の予約手続きに進むと在庫切れ等で先に進めないケースがある。
  これを事前に自動検出したい。
- 手動でのサイト操作・確認は時間がかかり、対象の航空会社やホテルが多いと現実的に全件を確認しきれない。
- SG（自社サイト相当）とTravelko（メタサーチ経由）で条件（サイトパラメータ）が異なるため、
  両方のパターンで動作を確認できるようにしたい。

## 3. できること（機能一覧）

- **自動収集**：出発地・目的地・出発日・現地出発日を指定するだけで、フライトとホテルの組み合わせを自動収集
- **複数の航空会社を自動巡回**：最安値の1件だけでなく、条件に応じて複数の航空会社を
  自動的に切り替えながら検証（同じ航空会社ばかりにならないよう、重複を自動的に間引くルール付き）
- **ホテルの取得条件を柔軟に指定**：「価格が安い順」「ホテルランクが高い順」それぞれ
  0〜3件まで、必要な件数だけ取得可能
- **予約直前画面への到達検証**：収集した組み合わせについて、実際に予約者情報入力画面まで
  到達できるかを自動チェックし、OK／NGで結果表示
- **キャンセル無料条件の一括検証**：キャンセル無料のプランのみを対象に、まとめて検証を実行
- **サイト切り替え**：SG／Travelkoの2パターンを切り替えて検証可能
- **ブラウザ画面（ローカルWeb UI）で結果を確認**：検証結果・関連URLを画面上で確認し、
  ワンクリックでコピー・別タブで開くことが可能

## 4. 全体の流れ

大まかな処理の流れは以下の通りです。

1. 条件（出発地・目的地・日程）を指定して検索
2. 該当する航空会社・フライトを自動選択（キャリアの巡回ルールに従って複数パターンを実施）
3. ホテルを指定条件（安い順／ランク順）で自動選択
4. 在庫確認API（実サイトの在庫確認処理）を実行
5. 予約者情報入力画面への到達を確認し、OK／NGを判定

この一連の流れを、条件を変えながら自動的に繰り返します。

## 5. 画面イメージ

- **検索フォーム**：出発地・目的地・日付・サイト（SG／Travelko）・ホテル取得件数を入力する画面
- **結果一覧**：航空会社ごとに見出しが分かれ、その下に価格順・ランク順それぞれの
  ホテル候補がカード形式で並ぶ
- **各カードの表示内容**：判定結果（未実行／実行中／OK／NG）、フライト情報
  （航空会社・便名・時刻・料金）、ホテル情報（ホテル名・料金・キャンセル可否）、
  到達確認後のURL（コピー可能）
- **一括操作**：「キャンセル無料の商品だけをまとめて検証」ボタンで効率的に確認可能

## 6. 技術構成（エンジニア向け）

**構成：**

- `reserve.js` — Playwright（ブラウザ自動操作ライブラリ）を用いたコマンドラインツール本体。
  サイトへの検索・選択操作、在庫確認APIの呼び出し、結果判定を行う
- `server.js` — Node.js標準機能のみで作成したローカルHTTPサーバー。
  ブラウザ画面からのリクエストを受け、reserve.jsを子プロセスとして実行し結果を返す
  （検証ロジックの二重管理を避けるため、判定ロジックは常にreserve.js側に集約）
- `public/` — ブラウザで表示するUI一式（HTML／CSS／JavaScript）

**主要な技術キーワード：**

- `airCacheKey` / `hotelCacheKey` / `selectedItemKey` —
  検索条件・選択商品をサーバー側で参照するためのキャッシュキー（Cookie不要で持ち運び可能）
- `packageId` / `__tx__` — 在庫確認後に発行される使い捨てトークン。
  予約者情報入力画面への遷移に必要
- **キャリア巡回ルール**：航空券一覧の1位（最安）は必ず検証対象とし、
  直前に検証した航空会社と同じ場合はスキップ、同一航空会社は最大2件まで、という条件で
  効率よく複数パターンを検証する

**重要な前提：**本ツールは実際のエアトリサイトへアクセスして検証を行うものであり、
モック環境ではありません。一方で、**予約の確定・決済処理は一切行いません**
（予約者情報入力画面が表示されるところまでを検証範囲としています）。

## 7. これまでの実績・検証状況

- 複数の出発地・目的地の組み合わせで、予約者情報入力画面への到達を確認済み
- 複数航空会社にまたがる自動巡回（同一キャリアの重複除外を含む）の動作を確認済み
- ホテルの「安い順」「ランク順」を個別件数指定で取得できることを確認済み
- キャンセル無料条件での一括検証が正しく対象を絞り込めることを確認済み
- SG／Travelko両サイトパターンでの動作確認（一部は検証継続中）
- ブラウザ画面からの一連の操作（検索→検証→結果確認）が問題なく動作することを確認済み

## 8. 制約・注意事項

- 実際のサイトへアクセスするため、対象件数が多い場合は検証に数十秒〜30分程度かかることがある
- 予約確定・決済機能は搭載しておらず、今後もその予定はない（検証専用ツール）
- 発行されるトークン（packageId／__tx__）は実行のたびに新しく発行される使い捨てのものであり、
  長時間経過すると無効になる

## 9. 今後の検討事項

- **航空会社の事前絞り込み**：特定の航空会社（例：ANAのみ）に絞って検索する仕組みが
  サイト側の機能として存在することを確認済み。必要であれば追加実装可能（現時点では未着手）
- 検証観点の拡充（表示価格と確定価格の差異検出など）について、必要に応じて継続検討
